# 🌐 Web Server Setup Notes (Linux + Windows)

---

## 🔹 1. What is a Web Server?

* A **web server** hosts and serves websites over the internet.
* Uses protocols:

  * **HTTP (80)** → Unsecured
  * **HTTPS (443)** → Secured (SSL/TLS)

---

## 🔹 2. Accessing Linux VM (Azure/AWS)

### ✅ Using SSH (Linux/macOS/WSL)

```bash
cd Downloads
chmod 400 web_key.pem
ssh -i web_key.pem azureuser@51.8.106.38
```

### 🔸 Important Points:

* `chmod 400` → Restricts key permissions (required for SSH)
* `azureuser` → Default username (Azure VM)
* Port used → **22 (SSH)**

---

### ✅ Using PuTTY (Windows)

#### Steps:

1. Download from 👉 PuTTY
2. Convert `.pem → .ppk` using **PuTTYgen**
3. Load `.ppk` in PuTTY
4. Enter:

   * Host: `public-ip`
   * Port: `22`
5. Connect

---

## 🔹 3. Apache Web Server Installation (Ubuntu)

### 📦 Install Apache

```bash
sudo apt update -y
sudo apt install apache2 -y
```

### ▶️ Start & Enable Service

```bash
sudo systemctl start apache2
sudo systemctl enable apache2
sudo systemctl status apache2
```

---

## 🔹 4. Web Server Directory

### 📁 Navigate to Default Web Root

```bash
cd /var/www/html
pwd
```

### ✏️ Modify Website

```bash
sudo rm index.html
sudo touch index.html
sudo nano index.html
```

### 🔸 Nano Editor Shortcuts:

* Save → `Ctrl + S`
* Exit → `Ctrl + X`

## 🔹 8. Remote Access Protocols

### 🔐 Linux Access

* Protocol: **SSH**
* Port: **22**
* Tool: Terminal / PuTTY

---

### 🖥️ Windows Access

* Protocol: **RDP (Remote Desktop Protocol)**
* Port: **3389**

### 📂 RDP Connection:

* Requires:

  * Public IP
  * Username
  * Password
* Open `.rdp` file or use Remote Desktop app

---

## 🔹 9. Quick Comparison

| Feature        | Linux VM       | Windows VM        |
| -------------- | -------------- | ----------------- |
| Access Method  | SSH            | RDP               |
| Port           | 22             | 3389              |
| Authentication | Key (.pem)     | Username/Password |
| Tool           | Terminal/PuTTY | Remote Desktop    |

* Manage servers (Linux/Windows)

* Monitor uptime & logs

## 🚀 Bonus Tips (Exam + Real-Time)

* Always allow **port 80/443 in NSG/Security Group**
* Use **HTTPS in production**
* Prefer **Nginx for high performance**
* Monitor using:

  * Prometheus + Grafana
  * Cloud-native tools (Azure Monitor / CloudWatch)
